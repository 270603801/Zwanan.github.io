part 1 数字三角形模型

数字三角形

​ 可以从上往下走，每层的状态由上一层得到，最后的答案为最底下的数据的最大值。

​ 也可以从下往上走，每层这个数由下面层得到。

最低通行费

\(2N - 1\)步就是不回头，从左上角只向右或者向下走

方格取数

从左上角到右下角，找到两条路的和最大。

状态: \(f[k][i1][i2]\) ，当前两条路都走了 \(k\) 步，走到 \([i1, k - i1],[i2, k - i2]\) ，需要判断是否重复的点。

四个状态:{ 第一条向下， 第二条向下: {

​ \(f[k - 1, i1 - 1, i2 - 1]\) \(+\) \(w[i1, j1]\) \(+\) \((重合 ? 0 : w[i2, j2]);\)

​ }, ​ 第一条向下， 第二条向右: {

​ \(f[k - 1, i1 - 1, i2]\) \(+\) \(w[i1, j1]\) \(+\) \((重合 ? 0 : w[i2, j2]);\)

​ }, ​ 第一条向右， 第二条向下: {

​ \(f[k - 1, i1, i2 - 1]\) \(+\) \(w[i1, j1]\) \(+\) \((重合 ? 0 : w[i2, j2]);\)

​ }, ​ 第一条向右， 第二条向右: {

​ \(f[k - 1, i1, i2]\) \(+\) \(w[i1, j1]\) \(+\) \((重合 ? 0 : w[i2, j2]);\)

​ }

part2 线性dp

最长上升子序列

两种解法:

  1. \(n^2\) 解法

    状态: \(f[i]\) 当前点为 \(i\) 的时候，最长的子序列长度

    当\(1,,,i - 1\) 中间有一个点小于当前点， 当前点 \(f[i]\) \(=\) \(max(f[i], f[j] + 1)\)

    否则 \(f[i]\) \(=\) \(1\)

  2. \(n*log(n)\) 解法

    贪心 + 二分， 使用单调队列\(f[len]\)。

    状态:

    当前点大于队尾\(f[len]\)的时候，直接加到队尾

    当前点小于队尾\(f[len]\)的时候，在当前队列中找出比当前点大的最小的数替换掉(使得队列的数字越小，数字加入的期望越高)

友好城市

拦截导弹

  • Dilworth 定理 ：

该定理断言：对于任意有限偏序集，其最大反链中元素的数目必等于最小链划分中链的数目。此定理的对偶形式亦真，它断言：对于任意有限偏序集，其最长链中元素的数目必等于其最小反链划分中反链的数目。

把该题第二问求，使用最少的系统，就是求最少需要几个不上升子序列

最少的不上升的数目换成求最大上升子序列的长度

最长公共子序列

简单版本\(O(n^2)\)，简单版本

\(f[i,j]\) 表示：当前 \(a\) 串遍历到 \(i\) ，\(b\) 串遍历到 \(j\) 时，最长的公共序列

状态:

\(f[i,j]= \begin{cases} max(f[i -1,j],f[i,j - 1]),&a[i] != b[i],\\ f[i - 1, j - 1] + 1, &a[i] == b[i] \end{cases}\)

困难版本\(O(nlog_2n)\)，困难版本

把二维化成一维，复杂度从\(O(n^2)\)化成\(O(nlogn)\)

状态: 假定2个串，因为其中的数字是完全相同的，所以可以采用离散化的方式

把第二个串中的大小按照在第一个串中的坐标设定

得出：第二个串有序的从前往后遍历，只要求最长上升子序列就可以了。就是说求得的该序列，在第一个中必然有序，由于在第二个中按照顺序遍历，也是有序的。

part3 背包

01背包

  • 朴素

\(f[n,v]\)表示，当前为第 \(n\) 个，且背包使用的容量为 \(v\) 的时候最大的价值

状态:

\(f[i, j] = \begin{cases} f[i - 1, j], \\ f[i - 1, j - v[i]] + w[i], & j >= v[i], \end{cases}\)

  • 优化

\(f[v]\)表示，当使用 \(v\) 的容量时最大的价值

状态: \(f[j] = max(f[j], f[j - v[i]] + w[i])\)

完全背包

优化过程:

\(O(nmk) >>> O(nm)\) && \(f[N][M] >>> f[M]\)

  • 最朴素
  • 优化时间
  • 优化内存

多重背包

优化过程:

\(O(nms)>>> O(nmlog_2s) >>> O(nm)\)

  • 朴素,朴素版

  • 二进制优化,困难版

  • 单调队列优化 困难版

混合背包

多种背包的情况，按照背包各自的情况做就行

二维费用背包

二维费用背包：有二维的因素决定价值

\(f[j, k]\) 表示当使用了 \(j\) 容量且当前重量为 \(k\) 的最大价值

状态:

\(f[j, k]\) \(=\) \(max(f[j, k], f[j - v[i],k - we[i]] + w[i]);\)

分组背包

分组背包: 有 \(n\) 组物品，组内物品互斥，只能选一个

  • 朴素

\(f[i][j]\) 表示 当前在第 \(i\) 组挑选，并且使用了 \(j\) 的容量的最大价值

状态:

\(f[i, j]\) \(=\) $max(f[i, j], f[i - 1, j - v[i][k]] + w[i][k]) $

  • 状态压缩

\(f[j]\) 表示 当前使用了$ j $ 容量时的最大价值

状态:

\(f[j]\) \(=\) \(max(f[j], f[j - a[i][k]] + w[i][k])\)

由朴素算法得到，每层的状态都会从本层和上一层得到，所以直接压缩掉一维状态

潜水员

也是类似的二维费用背包问题

\(f[j, k]\) 表示当使用不少于 \(f\) 容量的氧气和不少于 \(k\) 容量的氮气的时候的最小花费

状态:

\(f[j, k]\) \(=\) \(min(f[j,k],\) $f[max(0, j - o[i]), $\(max(0, k - n[i])] + w[i])\)

\(f[max(0, j - o[i]), max(0, k - n[i])]\) 中向上取整是因为可能有浪费的，不一定刚刚好用完
